## Vaulten v2.2.1

### Bug Fixes

- Fixed double-tap on search results opening the wrong credential — the first item in the filtered list would incorrectly open the credential that previously occupied that position before the search was performed. Root cause was a stale gesture closure in `pointerInput(Unit)` that never restarted when a different credential occupied the same card slot.
- Fixed Backup Preview page not being scrollable when the keyboard is open — passphrase field and action buttons could be obscured by the soft keyboard. Replaced incompatible `Modifier.weight()` spacers with fixed-height spacers to support vertical scrolling.

### Performance

- Optimized credential list rendering for large datasets (5000+ items) by adding stable `key` to `LazyColumn` items, enabling efficient diffing and minimal recomposition on filter/sort changes.
- Refactored dashboard ViewModel into a 2-phase pipeline: Phase 1 maps domain models to UI models only when DB data changes; Phase 2 handles filtering and sorting on pre-mapped data, avoiding redundant object allocations on every user interaction.
- Added 300ms debounce on credential search to prevent rapid sequential re-filtering on every keystroke.

### UX Improvements

- Added auto-dismiss keyboard on tap outside the search field on the Credentials Dashboard page.
- Added auto-dismiss keyboard on tap outside the search field on the Platforms Dashboard page.
- Added auto-dismiss keyboard on tap outside the passphrase field on the Backup Preview page.
