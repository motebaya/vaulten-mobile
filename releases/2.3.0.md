## Vaulten v2.3.0

### Major UI/UX Revision

This release replaces the hamburger menu navigation with a modern floating bottom navigation bar and introduces swipe-based page navigation.

#### Navigation Overhaul

- **Removed hamburger drawer menu** ŌĆö replaced with a floating pill-shaped bottom navigation bar positioned at the bottom center of the screen with spacing on all sides.
- **Floating bottom nav bar** features a semi-transparent glass-like appearance with semi-rounded capsule shape, shadow elevation, and smooth show/hide animation tied to scroll direction.
- **Nav items** (left to right): Credentials, Platforms, Settings, Support ŌĆö each with icon and label below. Active page icon and label are highlighted in the primary blue color.
- **Swipe gesture navigation** between all 4 main pages via HorizontalPager. Swiping right-to-left on the last page (Support) wraps around to the first page (Credentials).

#### Scroll Behavior

- **Scroll down** hides the floating nav bar and FAB with a smooth slide-out animation.
- **Scroll up** reveals them again with a smooth slide-in animation.
- Scroll direction detection uses a threshold-based accumulator to prevent jitter.

#### FAB Changes

- The "+" floating action button on Credentials and Platforms pages now animates in/out alongside the bottom nav bar based on scroll direction.
- FAB is positioned above the floating nav bar with proper spacing.

#### Lock Vault Relocation

- **Lock Vault** button moved from the bottom of the hamburger drawer to a **lock icon in the top-right corner** of every main page's top app bar, providing quicker access.

#### Search Field Styling

- Search fields on Credentials and Platforms pages updated from rectangular to **semi-rounded** input style (`RoundedCornerShape(24.dp)`) for a more modern look.

#### Top App Bar

- Each main page now shows the **app icon on the left**, **centered page title**, and **lock icon on the right** in the top app bar.

#### Architecture Changes

- Introduced `MainScreen.kt` ŌĆö a new pager container that hosts all 4 main screens, manages scroll state aggregation, nav bar visibility, and pager-to-nav synchronization.
- Introduced `FloatingBottomNavBar.kt` ŌĆö a new reusable floating pill navigation component.
- Introduced `PageScaffold` ŌĆö a lightweight scaffold replacement for pages hosted inside the pager (replaces the old `AppScaffold` with `ModalNavigationDrawer`).
- Consolidated 4 separate navigation routes (dashboard, platforms, settings, support) into a single `Main` route in `VaultNavHost`.
- Added `compose-foundation` dependency for `HorizontalPager` support.
- All 4 main screens (`DashboardScreen`, `PlatformScreen`, `SettingsScreen`, `SupportScreen`) had their signatures simplified ŌĆö removed `currentRoute` and `onNavigate` parameters since navigation is now handled by the pager.
- Added extra bottom padding (96dp) to scrollable content lists to prevent items from being obscured by the floating nav bar.
