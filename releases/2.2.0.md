## Vaulten v2.2.0

### Package Refactor

- Renamed package from `com.vaultapp` to `com.motebaya.vaulten` across the entire codebase
- Updated all 85 Kotlin source files, build configs, ProGuard rules, and resource references
- Renamed project from `SecureVault` to `Vaulten`
- Updated all branding strings, theme names, and User-Agent identifiers

### Build & Distribution

- Added per-ABI split APKs: `armeabi-v7a`, `arm64-v8a`, `x86_64`, and universal
- Each ABI variant receives a unique `versionCode` for Play Store compatibility
- Added GitHub Actions CI/CD workflow for automated build, tag, and release on push to main

### App Icon

- Replaced all launcher icons across all density buckets (ldpi through xxxhdpi)
- Updated adaptive icon foreground, round icons, and in-app branding icons

### Bug Fixes

- Restored missing `KeystoreManager.kt` — Android Keystore wrapper for hardware-backed key storage
- Handles device KEK generation, biometric key lifecycle, and AES-256-GCM wrap/unwrap operations
- Resolved `error.NonExistentClass` compilation failures caused by the missing file
